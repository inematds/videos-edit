🧱 FFmpeg: a base imprescindível

O FFmpeg é a ferramenta que corta e renderiza vídeo por baixo de tudo — é a base do Remotion, que o Hyperframes usa para montar o motion. É o único item verdadeiramente obrigatório do miolo: sem ele, nada roda. A instalação muda um pouco por sistema operacional.

Objetivo: instalar o FFmpeg no seu sistema. Escolha a linha do seu SO (as outras estão comentadas com #) e cole no terminal.

terminal · instalar FFmpeg 
# macOS (Homebrew)
brew install ffmpeg

# Ubuntu / Debian
sudo apt update && sudo apt install -y ffmpeg

# Windows (PowerShell, com winget)
winget install --id Gyan.FFmpeg -e

Como verificar

Rode o comando abaixo. Deve aparecer uma linha começando com ffmpeg version .... Se aparecer "comando não encontrado", reabra o terminal (ou a sessão) e tente de novo.

ffmpeg -version
O que é

A ferramenta de corte e render que é a base do Remotion/Hyperframes.

Por que aprender

É o único item realmente obrigatório; sem ele nada roda.

Conceitos-chave

FFmpeg, render, dependência de sistema.

2

⚙️ Hyperframes: o motor de motion

O Hyperframes é o segundo imprescindível: é o motor que monta as animações (motion graphics) do reel — texto que entra com punch, chips, reveals. Ele roda via npx, então não precisa de instalação pesada; o próprio comando baixa e executa na hora.

Novo aqui? — "npx"

npx é um atalho que vem junto do Node.js: ele executa um pacote sem você instalá-lo permanentemente. Na primeira vez pode baixar alguns arquivos — é normal e só acontece uma vez.

Objetivo: confirmar que o Hyperframes responde na sua máquina.

terminal · checar Hyperframes 
npx hyperframes --version

Como verificar

O comando deve imprimir um número de versão (algo como 2.x.x). Se ele perguntar para instalar o pacote na primeira vez, aceite. Se der erro de npx não encontrado, instale o Node.js antes.

O que é

O motor que monta o motion do reel, rodando via npx.

Por que aprender

Sem ele não há motion graphics; um comando confirma que está no ar.

Conceitos-chave

Hyperframes, npx, motion graphics.

3

🧠 Groq: a revisão que menos inventa

O Groq é grátis e muito recomendado para a transcrição (base das legendas) e para a revisão — na prática, é o que menos "inventa" nessa tarefa. Não precisa instalar nada: basta ter a chave de API disponível como variável de ambiente chamada GROQ_API_KEY.

Objetivo: checar se a chave já está no ambiente e, se não estiver, defini-la. Troque o valor de exemplo pela sua chave real.

terminal · GROQ_API_KEY 
# 1) já está definida? (deve imprimir a chave)
printenv GROQ_API_KEY

# 2) se não imprimiu nada, defina (troque pela sua chave)
export GROQ_API_KEY="<sua-chave-gsk_...>"

Como verificar

Depois do export, rode printenv GROQ_API_KEY de novo: agora deve imprimir a sua chave (começando com gsk_). Para não repetir a cada sessão, coloque o export no seu ~/.bashrc ou ~/.zshrc.

Por que aprender: uma variável de ambiente vazia é o erro silencioso mais comum aqui — o reel roda, mas as legendas saem piores. Checar antes evita retrabalho. Conceitos-chave: Groq, GROQ_API_KEY, variável de ambiente.

4

➕ Opcionais: fal.ai, Firecrawl, /watch

Estes três não são obrigatórios — você liga só o que o seu perfil pedir, e nada pago entra sem o seu "sim". Se faltar algum, o motor degrada com elegância em vez de travar.

🖼️

fal.ai (imagem por IA · pago)

Gera imagens para cold opens e insertos. Como é pago, só entra se você aprovar explicitamente.

🌐

Firecrawl (B-roll real · grátis)

Traz capturas reais de sites/repositórios que você cita, para usar como B-roll de verdade em vez de imagem genérica.

👁️

/watch (QC visual)

Faz o controle de qualidade olhando os fotogramas do reel — ajuda o revisor a pegar lacunas e sobreposições.

Objetivo: se o seu perfil pedir fal.ai e/ou Firecrawl, defina as chaves (opt-in). Pule as que não usar.

terminal · chaves opcionais 
# só se o seu perfil usar imagem por IA (pago)
export FAL_KEY="<sua-chave-fal>"

# só se o seu perfil usar B-roll real (grátis)
export FIRECRAWL_API_KEY="<sua-chave-firecrawl>"

Como verificar

Confirme com printenv FAL_KEY ou printenv FIRECRAWL_API_KEY. Não definiu nenhuma? Sem problema — o diagnóstico de ambiente (módulo 3.2) só vai pedir o que o seu perfil realmente precisar.

Conceitos-chave: fal.ai, Firecrawl, /watch, opt-in (você aprova antes de ligar).

5

📦 Instalar a .skill

Com o stack no lugar, falta carregar a meta-skill no Claude. O arquivo tem extensão .skill. Há dois caminhos — use o que for mais simples pra você.

⬇️ Baixar forja-reel.skill

O arquivo fica em download/forja-reel.skill na raiz do curso.

Caminho A — duplo-clique

Depois de baixar, dê duplo-clique no arquivo forja-reel.skill. O Claude reconhece e oferece carregar a habilidade.

Caminho B — pelos conectores

  1. Abra o Claude.
  2. Vá em conectores → skills.
  3. Escolha carregar habilidade e aponte para o .skill.

Como verificar

Na lista de skills do Claude deve aparecer a habilidade do Forja Reel disponível. Se não apareceu, confira que baixou o arquivo inteiro (não a página) e repita o Caminho B.

Conceitos-chave: arquivo .skill, conectores, carregar habilidade.

6

▶️ Ativar com "Forja Reel" e usar Opus

Skill carregada, é só chamar. Você ativa a meta-skill escrevendo "Forja Reel" na conversa com o Claude. Um detalhe que faz diferença: use o modelo Opus, não o Sonnet — a entrevista e as decisões de identidade ficam bem mais consistentes.

Objetivo: iniciar a meta-skill. Com Opus selecionado no Claude, mande esta mensagem.

Claude (modelo: Opus) · mensagem 
Forja Reel

Como verificar

A skill responde iniciando a entrevista — a primeira pergunta dos 8 blocos aparece. Se em vez disso vier uma resposta genérica, confirme que a skill está carregada (tópico 5) e que você escreveu o nome exato.

Por que Opus e não Sonnet

A entrevista pede raciocínio para recomendar boas escolhas de identidade e montar o perfil.json. O modelo mais forte conduz melhor essa conversa e erra menos nas decisões — vale trocar antes de começar.

Conceitos-chave: ativação, Opus, entrevista. No módulo 3.2 a gente entra na entrevista em detalhe.
